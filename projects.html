Skype is a VoIP messenger that enables the world to communicate effortlessly. Millions of individuals and businesses alike use Skype to make free video and voice calls. Skype also allows to send instant messages and share files with other people.

While working at Skype headquarters, I was improving their advertising rotation app as well as underlying infrastructure. Skype client used the Internet Explorer container to show ads. Since the version of IE varied from client to client, Skype faced multiple issues including memory leaks and excessive resource consumption. I helped optimize Skype’s ad rotation app, which resolved all of the known issues. In addition, I created complete test coverage of the system, which included both unit and functional tests. This eliminated the human factor of mistakes on deployment.

Moreover, I worked on Skype’s ad error analytics. Issues from clients were streamed to Hadoop cluster. Retrieving data from cluster required to write complex SQL queries, which sometimes took hours to execute. I was able to create a simple, yet powerful analytics panel. Since then, everyone on the team had instant access to real-time error reports. This required no SQL knowledge and the results were shown as comprehensive interactive charts. I’ve also worked on Skype’s chat bots, handling over 50,000 simultaneous users and created a few R&D prototypes.

My solution reduced the team reaction time significantly, which had a direct positive impact on business.

After my contract had come to an end, I went back home.

Profusa is a company, creating tissue-integrating biosensors for continuous contactless monitoring of blood chemistries. Some of the beneficiaries of Profusa include athletes, soldiers, and diabetics. Medical institutions are also using Profusa sensors for live monitoring the oxygen and sugar levels.

In order to fine-tune sensors during development, Profusa needed a system for real-time data collection, analysis, and consolidation. Their legacy system was not adequate to handle the task, so my team and I took charge of their legacy system modernization.

I was a Manager and a Team Lead for this project. Having started with only 3 people on the team, I was able to made thorough analysis of all issues that affected system performance and stability.

The challenge was to collect data from multiple sources like Bluetooth sensors, USB Sensors, and CSV-files. Since every source had it’s own data structure, the first thing we had to implement was a data consolidation algorithm. As a result, no matter the input source, all of the data had been converted to a single internal format. This significantly reduced code duplication and improved maintainability.

Performance and stability of chart rendering was another issue we had addressed. On large data sets, the chart was freezing, while on medium data sets, it would take a number of minutes for the chart to display. Benchmark clearly showed the problem existed on multiple levels. We had replaced the charting component, enabled the network traffic compression, and completely reworked the data pre-processing algorithms. This increased chart display speed by 40 times and resolved all known stability issues.

After the code and performance issues were addressed, we switched to developing the new features. We created a convenient web interface for importing data files, which supported multi-file uploads, had progress indication, and drag-n-drop support. This allowed anybody on the customer team to import arbitrary data generated by 3rd party software. Unfortunately, some of the improvements were not possible to implement in the scope of the existing architecture and technology stack. Therefore, we agreed to make a short proof of concept project intended to demonstrate future possibilities and become the base for the future versions.

The prototype we’ve created had a significantly faster data streaming based on Web Sockets, dynamic buffering to handle temporary disconnections, and supported pivoting displays for displaying charts in a stack.

We were able to finish our work before the due date, so we spent the remaining time writing a technical documentation, improving development infrastructure, and setting continuous delivery / integration.

Project has been successfully delivered and the client was very happy with the work we had done.

Softonic.com is a software and app discovery portal based in Barcelona. It is owned by Softonic International. It reaches over 100 million users per month. Softonic’s private network had dozens of services with an unrestricted access to each other, and needed an authorization system to resolve this problem.

My team and I chose to base the solution on OAuth 2.0 specification. It was a known standard that matched our goals perfectly, but didn’t address high-load issues. No physical server would stand authenticating all request in the network.

In order to resolve this problem I proposed to use asymmetric cryptography tokens with caching and timestamp-based invalidation. This idea appeared to be the best solution and had received approval by the client.

Having a few hundred total endpoints, Softonic needed a method for managing permissions. I had created a convenient admin panel for managing access control lists (ACLs). The panel included permission management and import/export utility.

After two months of development, the project had been successfully handed over on time.
